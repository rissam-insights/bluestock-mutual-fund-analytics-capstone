## 📌 Executive Summary

> **Bluestock Mutual Fund Analytics Capstone v1.0** is a comprehensive, end-to-end data analytics project that simulates the workflow of a professional financial analyst — from raw data ingestion to a fully interactive Power BI dashboard and a fund recommendation engine.

This capstone demonstrates mastery across the full data analytics lifecycle:

- 🗄️ **Data Engineering** — ETL pipeline using Python and SQLite
- 🔍 **Exploratory Analytics** — Statistical profiling and investor behavior analysis
- 📐 **Quantitative Finance** — CAGR, Sharpe, Sortino, Alpha, Beta, VaR, CVaR
- 📊 **Business Intelligence** — 4-page Power BI dashboard with executive KPIs
- 🤖 **Recommendation Engine** — Rule-based fund recommendation system (`recommender.py`)
- 📋 **Professional Reporting** — Final PDF report and executive presentation

**Business Problem Addressed:** Retail mutual fund investors lack accessible, data-driven tools to evaluate fund performance, manage risk exposure, and make informed investment decisions. This project builds exactly that.

## ⚙️ ETL Pipeline

<details>
<summary><b>Click to expand ETL details</b></summary>

```python
# Simplified ETL Pipeline Overview

# Stage 1 — EXTRACT
raw_nav     = pd.read_csv("data/raw/fund_nav.csv")
raw_txns    = pd.read_csv("data/raw/investor_transactions.csv")
raw_funds   = pd.read_csv("data/raw/fund_master.csv")

# Stage 2 — TRANSFORM
# ▸ Parse dates, standardise dtypes
# ▸ Remove duplicates & null NAV rows
# ▸ Compute daily returns: (NAV_t / NAV_t-1) - 1
# ▸ Merge fund master for category labels
# ▸ Engineer SIP streak features per investor

# Stage 3 — LOAD (SQLite)
conn = sqlite3.connect("data/db/bluestock_mf.db")
clean_nav.to_sql("fund_nav",          conn, if_exists="replace")
clean_txns.to_sql("transactions",     conn, if_exists="replace")
clean_funds.to_sql("fund_master",     conn, if_exists="replace")
```

## 📐 Analytics Performed

<details open>
<summary><b>📈 Performance Metrics</b></summary>

| Metric | Formula | Purpose |
|--------|---------|---------|
| **CAGR** | `(End NAV / Start NAV)^(1/n) − 1` | Long-term growth rate |
| **Sharpe Ratio** | `(Rp − Rf) / σp` | Risk-adjusted return |
| **Sortino Ratio** | `(Rp − Rf) / σ_downside` | Downside-only risk penalty |
| **Alpha (α)** | `Rp − [Rf + β(Rm − Rf)]` | Manager skill vs benchmark |
| **Beta (β)** | `Cov(Rp, Rm) / Var(Rm)` | Market sensitivity |
| **Max Drawdown** | `(Peak − Trough) / Peak` | Worst peak-to-valley loss |

</details>

<details>
<summary><b>⚠️ Risk Metrics</b></summary>

| Metric | Method | Confidence |
|--------|--------|-----------|
| **Historical VaR** | Percentile of return distribution | 95% |
| **CVaR / Expected Shortfall** | Mean of losses beyond VaR threshold | 95% |
| **Rolling 90-Day Sharpe** | 90-day rolling window Sharpe | Dynamic |

```python
# VaR & CVaR Computation
returns = fund_nav['daily_return'].dropna()
VaR_95  = np.percentile(returns, 5)
CVaR_95 = returns[returns <= VaR_95].mean()
```

</details>
